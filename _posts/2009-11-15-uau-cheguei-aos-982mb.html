Sempre fui poupado, forreta, mão de vaca, fuinha ou como quiserem chamar. Isto não só no que diz respeito a dinheiro, mas a tudo. Mesmo TUDO. Uma delas é na utilização dos recursos do meu computador. Mesmo nestes últimos 2 anos em que passei a ter 2 Gb de RAM disponíveis, esforço-me por manter todos os serviços Windows dispensáveis desligados. O estado da memória do meu computador após o arranque é de 173 Mb. Isto porque nestes 2 anos lá tive que instalar alguns drivers adicionais que fizeram aumentar o consumo de memória inicial dos 157Mb, para os 173 Mb.  

 

Este meu espírito de poupança (ou maluquice) tem sido uma das razões que me fez adiar o upgrade para o Windows Vista. Além de que gosto de olhar para o Task Manager e identificar num ápice os 22 processos que tenho a correr.

 

Por isso é que quando oiço falar naquelas tangas do Linux, um sistema operativo ‘levezinho’, freeware, open source, bla bla bla,, só me dá vontade de rir. Infelizmente também tenho que trabalhar no Linux, neste caso a distribuição do Ubuntu e, só no estado inicial tenho 256 Mb de memória logo ocupados e facilmente chego a 1 Gb usando IDEs como o Eclipse ou o Netbeans.

 

Mas o inédito aconteceu esta semana quando a trabalhar no Windows atingi 982 Mb. Segue o printscrean dos maiores esbanjadores: Servidor Web ASP.Net WebDev (81), Sql Server (56), 2 x VisualStudio (38+ 64), Winword (35), power point (31), 2 x IE (35+40), Msn Messenger + Wlcomm(48+35). Tudo tolerável, com excepção deste último que é um autêntico abuso – 83 Mb no total. Estou a preparar um substituto para ele.

 

 

P.S. Um dia destes, só para me rir mais um pouco, dou-me ao trabalho de fazer um printscreen do Task Manager (ou lá como se chama) do Ubuntu, com duas instancias de Elipse no ar, o MySql, um servidor web, Open Office Word e power point. Vai ser um fartote.
